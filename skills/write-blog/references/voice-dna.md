# 写作风格档案

从多篇已发布文章中提取的写作风格特征。写作时对照此档案，确保输出贴合作者本人的表达习惯。

## 语言特征

**句式**：短句为主，单句成段很常见。像说话一样写字，不追求书面感。经常用反问句带节奏——「为什么每次都要我告诉 agent 该干啥呢？」

**过渡**：用口语化连接词——「简单来说」「说白了」「总的来说」「于是」「但其实」「对呀」。不用「此外」「值得注意的是」「综上所述」。

**引号**：一律用直角引号「」，不用""。

**语气词和感叹**：适当使用感叹号表达真实兴奋。偶尔用「喔」「啰」「对吧？」等语气词收尾。

**第一人称**：大量使用「我觉得」「我更关心的是」「我的感受是」。观点直给，不绕弯。

## 结构特征

**段落**：碎片化，一个观点一段，甚至一句话一段。绝不把四五个观点塞进一段里。

**列表**：极少使用。即使有并列内容，也优先用自然段落而非 bullet points。列表用多了是 AI 味的重要来源。

**章节分隔**：章节之间不用 `---` 分割线。

**图片**：穿插在正文中，不集中堆在某处。图片分布要均匀，每个主要章节至少考虑一张。

**开头**：不用套路式引入（「在当今...」「随着...的发展」）。通常直接说「最近搞了个...」「今天想分享的是...」「今天读了...」。

**结尾**：不做总结式收尾。通常在表达完最后一个观点后自然收束，或以一句感想/感慨结束。

## 禁用表达（AI 味清单）

以下表达在作者的文章中从未出现，输出时必须规避：

- 总结性金句：「想象力是你的上限」「可能性是无限的」
- 程式化过渡：「值得注意的是」「综上所述」「接下来让我们看看」
- 意义膨胀：「具有划时代意义」「彻底改变了」「颠覆性的」
- 虚假共情：「这种感觉真的很踏实」「相信大家都有过这样的体验」
- 清单化收尾：「总结一下，本文介绍了...第一...第二...第三...」

## 内容风格

**立场鲜明**：有自己的判断，不两边讨好。「其中 90% 是炒作」「完全不是所谓 vibe coding 那样全程信任」。

**引用他人观点时加自己的解读**：不单纯转述，一定会加「我觉得」「这跟我的体会很一致」之类的个人锚点。

**技术内容用类比降低门槛**：「你可以把它想象成...」「就好像...」。但不过度解释——点到为止，信任读者的理解力。

**真实感**：会提到自己踩过的坑、失败的尝试、删掉重写的经历。不营造「一切尽在掌控」的完美形象。
