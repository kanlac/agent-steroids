# Write Blog

将录音稿、口述笔记或零散想法转化为符合作者风格的博客文章。

## 写作风格

读取 [references/voice-dna.md](references/voice-dna.md) 了解作者的完整写作风格档案，包括语言特征、结构偏好、禁用表达。**每次写作前必须读取。**

## 工作方式

**理解输入**：录音稿是文章的灵魂——里面有作者的真实想法、情绪、结构倾向。错别字和口误不重要，重要的是意会作者想表达什么、想按什么顺序讲。如果用户同时指向了技术素材（代码仓库、文档、项目目录），先完整阅读这些素材，理解细节后再动笔。

**图片规划**：写作时同步规划图片插入位置，用 `> 📌 图片建议：...` 标注。图片要均匀分布在各章节，避免集中堆放。图片类型包括：截图、对比图、代码截图、产品界面、第三方生成的插图等。

**自检**：初稿完成后，对照 voice-dna.md 中的禁用表达清单和结构偏好做一轮自检，修正后再呈现给用户。

## 受众意识

文章面向混合受众——有技术背景的读者和非技术读者并存。技术概念需要用类比解释清楚（「你可以把它想象成...」），但点到为止，不过度展开成教程。信任读者的理解力。

## Frontmatter

文章使用 Obsidian frontmatter：

```yaml
---
writing: true
publish-date: YYYY-MM-DD
---
```
