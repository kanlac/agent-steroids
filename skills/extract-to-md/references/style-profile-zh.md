# 中文排版偏好

以下规则用于报告类 Markdown 的统一整理与润色。

## 标点与引号

- 引号优先使用直角引号：`「」`，必要时使用 `『』`。
- 数值小数必须使用半角点号：`1.56`、`6.7`、`3.4`。
- 摘要编号使用 Markdown 标准形式：`1.`、`2.`、`3.`。
- 中文与英文之间需加空格（例如：`中文 English`）。
- URL、邮箱、代码片段中的标点保持半角，不做全角替换。
- 若用户要求全角标点，也不得改动 URL 内部符号。

## 段落与加粗

- 段首不加空格，不使用首行缩进。
- 段内错误换行应合并；仅在真实段落边界处换段。
- 加粗范围必须严格按用户指令执行（例如仅标题加粗）。

## 图片与位置

- 图片应尽量贴近原 PDF 的上下文位置插入。
- 图表类图片不要集中堆到文末。
- 图片统一导出到独立资源目录，并在 Markdown 使用相对路径引用。
